How Does Javascript Works!!!

JavaScript is a scripting language for the web and one of the most efficient, commonly used scripting languages. It is an interpreted language, which means it does not need a compiler to translate its code like C or C++. JavaScript code runs directly in a web browser.JavaScript is asynchronous by nature, meaning it is designed to ensure the non-blocking code execution. Non blocking code do not prevent the execution of another piece of code. This means that the server works in much the same way, waiting in a loop for a network request, and accepting more incoming requests while the first one is being handled.The way JavaScript works is interesting. Inside a normal Web page we place some JavaScript code. When the browser loads the page, the browser has a built-in interpreter that reads the JavaScript code it finds in the page and runs it.

Difference between local storage and session storage

LocalStorage: it's called it's local storage for your browsers, it can save up to 10MB. localStorage persists the storage across browser sessions. if opened a new tab, or the browser is closed and reopened, the storage is saved. There is no expiration.This read-only interface property provides access to the Documentâ€™s local storage object, the stored data is stored across browser sessions. The data stored in localStorage persists until explicitly deleted. Changes made are saved and available for all current and future visits to the site.

sessionStorage: SessionStorage also saved data, but as it's name saying, it's session based and will be deleted after closing your browser, also can save less than LocalStorage, like up to 5MB, Session Storage objects can be accessed using the sessionStorage read-only property. The data is available only inside the window/tab in which it was set. sessionStorage is cleared when the page session ends. A page session lasts as long as the tab or the browser is open. Therefore, when you open a new tab or a new window, this creates a new session and new storage.
